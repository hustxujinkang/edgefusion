# 台区智能融合终端后台系统架构设计

## 1. 系统概述

台区智能融合终端后台系统是一个基于Python的后台程序，用于对台区内的光伏、储能、充电桩等设备进行协同控制和监控。系统采用模块化设计，具有良好的扩展性和灵活性。

## 4. 核心功能设计

### 4.1 型号点表系统

系统支持通过点表（Point Table）定义不同型号设备的寄存器映射：

```python
POINT_TABLES = {
    'xj_dc_120kw': {
        'protocol': 'modbus_tcp',
        'max_power': 120,
        'gun_count': 2,
        'registers': {
            'pile': {...},    # 桩信息区 0x1000
            'gun1': {...},    # 枪1数据区 0x2000
            'gun2': {...},    # 枪2数据区 0x2100
        },
        'control': {...}      # 控制命令定义
    }
}
```

点表支持：
- 不同地址区域的定义（桩信息、枪数据、控制区）
- 数据类型映射（u16, i16, u32, i32, float32）
- 缩放因子和偏移量
- 单位标注

### 4.2 统一控制映射

系统通过 `point_tables.py` 和 `adapters/device_profiles.py` 将不同型号设备展开为统一的语义采集点和控制命令，再由 `DeviceManager`、协议层和传输层执行真实读写。

统一控制链路的特点：
- 业务层和监控面板都只使用语义字段和语义命令
- 点表和适配层负责把型号差异翻译成 `telemetry_map` / `control_map`
- 协议层负责 Modbus/MQTT/OCPP 等协议语义
- 传输层负责 TCP 等物理连接承载
- 充电桩按“桩接入、枪控制”建模，策略控制和人工控制共用同一条下发链路

### 4.3 设备管理

- **设备注册**：支持手动注册和自动发现
- **协议适配**：支持Modbus TCP/RTU、MQTT、OCPP等多种协议
- **型号识别**：通过配置的型号自动加载对应点表
- **设备状态**：实时监测设备运行状态
- **设备控制**：统一的设备控制接口

### 4.4 控制策略

- **策略配置**：支持通过配置文件或API配置策略
- **策略引擎**：基于规则的策略执行引擎
- **协同控制**：多设备协同工作的智能调度
- **策略模板**：内置常用策略模板

### 4.5 监控系统

- **数据采集**：实时采集设备运行数据
- **数据存储**：本地存储和远程上传
- **监控面板**：Web界面展示系统状态
- **充电枪监控**：实时显示各枪状态和参数
- **告警管理**：异常情况及时告警

## 5. 系统流程

1. **系统启动**：加载配置，初始化各个模块
2. **设备发现**：自动或手动发现并注册设备
3. **点表加载**：根据设备型号加载对应的点表配置
4. **策略加载**：加载并初始化控制策略
5. **数据采集**：定期采集设备运行数据
6. **策略执行**：根据采集的数据执行控制策略
7. **监控展示**：实时展示系统运行状态
8. **异常处理**：处理设备异常和系统故障

## 6. 关键设计决策

### 6.1 型号配置而非自动识别

系统采用手动配置设备型号的方式，而非自动识别：
- **优点**：配置明确，避免识别错误，支持更多型号
- **实现**：设备添加时选择型号，存储到数据库

### 6.2 渐进功率调节

模拟器和统一控制链路都围绕渐进功率调节设计：
- **原因**：保护电池和电网，模拟真实设备行为
- **速率**：3kW/s 的调节速率

### 6.3 控制区与数据区分离

120kW/240kW型号采用独立的控制寄存器区：
- **控制区 (0x4000)**：接收控制命令
- **数据区 (0x2000/0x2100)**：上报枪状态
- **内部区 (0x3000)**：存储控制状态

### 6.4 双枪独立控制

双枪充电桩的两枪完全独立：
- 独立的状态监控
- 独立的功率控制
- 可同时充电，功率限制分别生效
